# Автотесты для Orbita Market
## Запуск
### Запуск тестов
 * Убедиться, что тестируемое приложение запущено
 * Перейти в корень проекта автотестов
 * Выполнить команду ```mvn test```
 * После завершения тестов выполнить команду ```mvn allure:serve```
для генерации и просмотра отчёта

### Запуск тестируемого приложения
 * Перейти в корень проекта тестируемого приложения
 * Создать файл .env и определить в нём переменные окружения
по примеру из файла .env.example
 * выполнить команду ```docker compose up --build```

## Чек-лист
### Тестирование базовых сценариев
1. Happy Path
   * Запрос на создание счёта. Статус-код 200;
   * Запрос на пополнение 1000 геокредитов.
    Проверка ответа: баланс=1000. Статус-код 200;
   * Запрос на создание заказа. Цена: 120 геокредитов;
   * Проверка ответа в течении 10 сек: баланс=880;
2. Недостаточно средства
   * Запрос на создание счёта. Статус-код 200;
   * Запрос на пополнение 50 геокредитов.
   Проверка ответа: баланс=50. Статус-код 200;;
   * Запрос на создание заказа. Цена: 120 геокредитов;
   * Запрос на получение баланса. Проверка ответа в течении 10 сек:
   баланс=50;
   * Запрос на получение информации о заказе. Проверка ответа
   в течении 10 сек: status=PAYMENT_FAILED
3. Дублирование события OrderPaymentRequested
    * Запрос на создание счёта. Статус-код 200;
    * Запрос на пополнение 500 геокредитов.
      Проверка ответа: баланс=500. Статус-код 200;
    * Запрос на создание заказа. Цена: 100 геокредитов;
    * Публикация события в Kafka с order_id как в последнем запросе;
    * Запрос на получение баланса. Проверка ответа:
      баланс=400;
4. Несколько заказов
    * Запрос на создание счёта. Статус-код 200;
    * Запрос на пополнение 1000 геокредитов.
      Проверка ответа: баланс=1000. Статус-код 200;
    * Запрос на создание заказа. Цена: 400 геокредитов;
    * Запрос на создание заказа. Цена: 400 геокредитов;
    * Запрос на получение баланса. Проверка ответа:
      баланс=200;
5. Повторное создание счёта
    * Запрос на создание счёта. Статус-код 200;
    * Запрос на пополнение 500 геокредитов;
    * Запрос на создание счёта с тем же user_id;
    * Проверка ответа: user_id совпадает, account_id совпадает
    с ранее созданным, balance=500
### Тестирование позитивных запросов к REST-эндпоинтам
#### Orders Service
1. POST /orders
    * Запрос на создание заказа;
    * Проверка ответа: статус-код 201;
    * Проверка ответа: status=CREATED, failure_reason=null,
    поля ответа соответствуют полям тела запроса;
2. GET /orders
    * Запрос на получение списка заказов;
    * Проверка ответа: статус-код 201;
    * Проверка ответа: первая запись списка соответствует
    последнему созданному заказу (п.1);
3. GET /orders/{order_id}
    * Запрос на получение последнего созданного заказа (п.1);
    * Проверка ответа: статус-код 201;
    * Проверка ответа: поля ответа соответствуют полям
    тела запроса (п.1);
#### Payments Service
1. POST /payments/accounts
   * Запрос на создание счёта. Статус-код 200;
   * Проверка ответа: balance=0;
2. POST /payments/top-up
   * Запрос на пополнение 500 геокредитов.
   * Проверка ответа: balance=500, статус-код 200,
   user_id как в запросе, currency=geocredits;
3. GET /payments/balance
   * Запрос на получение баланса
   * Проверка ответа: balance=500, статус-код 200,
     user_id как в запросе, currency=geocredits;
